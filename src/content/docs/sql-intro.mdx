SQL ist die Standardsprache, um mit relationalen Datenbanken zu "sprechen". Es ist wie die Grammatik und das Vokabular, das dein DBMS versteht.

Egal ob du MariaDB, MySQL, PostgreSQL, MS SQL Server oder Oracle verwendest – sie alle verstehen SQL (mit kleinen Unterschieden im Dialekt).

## SQL-Teilbereiche

SQL wird in mehrere Teilbereiche unterteilt:

### DDL (Data Definition Language)

Definiert die Struktur. Alles mit `CREATE`, `ALTER` (ändern), `DROP` (löschen).

*Der Architekt, der die Wände des Aktenschranks einzieht.*

### DML (Data Manipulation Language)

Manipuliert die Daten. Alles mit `INSERT`, `UPDATE`, `DELETE`.

*Der Sachbearbeiter, der Akten einfügt, aktualisiert oder vernichtet.*

### DQL (Data Query Language)

Frägt Daten ab. Der wichtigste Befehl: `SELECT`.

*Der Chef, der einen Bericht anfordert: "Gib mir alle Akten..."*

### DCL (Data Control Language)

Regelt Berechtigungen. `GRANT` (erlauben), `REVOKE` (entziehen).

*Der Sicherheitsdienst, der bestimmt, wer welche Tür aufsperren darf.*
